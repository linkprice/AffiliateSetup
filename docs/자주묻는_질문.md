## 실적 조회 API 

### Q. 구매/취소 확정 처리 시점 문의

* 매월 6일 전전월 정산이 완료되며 이후 확정 됩니다.
* 해당 일 이전 정산 기간 중 실적 데이터가 지속적으로 변경 되기 때문에 매월 6일 정산 완료 이후 에 실적 조회 API를 이용하여 조회 하실 것을 권장 드립니다.

### Q.  API 호출 제한 문의

* 별도의 호출 제한은 존재하지 않으나 월별 조회 등 시스템 부하가 발생할 수 있어 내부적으로는 15분에 1회 가량 호출하는 것을 권장하고 있습니다.
* 사전 협의 없이 자주 호출하여 시스템 부하 발생 시 별도 사전 안내 없이 해당 어필리에이트 코드 기준으로 API 호출이 금지 될 수 있으니 이용에 유의 부탁 드립니다.

### Q. 유니크 값 문의

* `trlog_id` 값은 내부적으로 관리되는 유니크 값입니다. 하지만 머천트별 정산 컨디션에 따라 `실시간 실적 전체 취소 후 구매 확정 실적 재생성` 되는 경우 리워드 API를 통해 status `310`으로 취소된 `trlog_id`로 status `210`으로 구매 확정 된 실적 추적 불가능합니다.
* 실적 추적 시 변하지 않는 값은 `m_id`, `o_cd` 입니다.

### Q. 응답 데이터 문의

* `status`

  * 100: 실적 발생 시 적재된 상태입니다.

  * 200: 정산 대기 상태. 내부적으로 정산 과정에서 관리되는 상태로 구매 확정인 상태가 아닙니다.

  * 210: 정산 완료 상태. 정산이 완료된 상태입니다.

  * 300: 취소 대기 상태. 내부적으로 정산 과정에서 관리되는 상태로 취소 확정된 상태가 아닙니다.

  * 310: 취소 확정 상태: 취소가 확정된 상태입니다.

  * 일부 머천트의 경우 매월 6일 이전 정산 기간 중에는 확정된 실적이 취소 상태로 조회 될 수 있사오니 연동에 참고 부탁드립니다.

* `create_time_stamp`

  * 실적이 링크프라이스 데이터베이스에 생성된 일자로, 실제 실적 발생일과 무관합니다.
  * 실제 실적 발생일은 `yyyymmdd`입니다.

* `pur_rate`

  * 적용된 머천트 프로그램의 건당 커미션 금액 또는 커미션 비율이며, 별도 프로모션에 의한 커미션 상향 값은 포함되지 않습니다.
  * 실제 `sales` 대비 내부적으로 계산되어 지급되는 커미션은 `commission` 항목에서 확인 가능합니다.

### 파라미터 문의

* 정산 완료, 취소 완료, 발생 실적 조회 시 조회 기준 일자는 실적 발생 기준일자입니다. 정산 완료,.취소 완료 시점에 대해서는 확인이 불가능합니다.

## 리워드 API

### 머천트별 실적 전송 방식

* 머천트별 실적 전송 방식의 경우 [광고주 조회 API](https://github.com/linkprice/AffiliateSetup/blob/master/docs/광고주_조회_오픈_API_1.0.md#추가-정보-응답-샘플) 의 `추가 정보 응답 샘플` 항목의  `when_trans` 응답 데이터 확인 바랍니다.
